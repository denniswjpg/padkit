Six keys and a knob that do exactly what you say.

PadKit is open firmware for the cheap macropad sold under a dozen different names. Flash it from this page in about a minute, then give every key its own colour and its own shortcut. The settings live on the pad itself, so they follow it to any computer.

Is this your pad?

Six keys, one knob, a CH552 inside.

The reference unit was sold as a "SinLoon Mini Macro Programmable Keyboard, 6 Keys 1 Knob". The same board turns up on AliExpress and eBay as a generic knob macropad. If yours matches the photo and reports the USB id below, PadKit runs on it.

Getting there

Three stages, and you can stop after the second.

Flashing is a one-off. After that everything happens in the browser, and the daemon is only worth installing if you want keys to run things on your machine.

01

Flash the firmware

Your browser talks to the chip's built-in bootloader over WebUSB. That bootloader sits in read-only ROM, so a failed flash is always recoverable.

Start flashing Guided, about a minute
02

Pick colours and keys

Choose a colour and a shortcut for each key, and for each direction of the knob. Everything is written straight to the pad and survives unplugging.

In practice

What people put on the keys.

The firmware sends F13 to F23, which no application claims by default. Bind them in macOS, Windows, Linux, AutoHotkey, Karabiner, or Home Assistant, the same way you bind any other key.

  • Calls. Mute and unmute on a key, camera on another, volume on the knob.
  • Media. Play, pause and skip on the keys, scrub or change volume with the knob.
  • Coding. Run the tests, format the file, stage and commit without leaving the keyboard.
  • Home Assistant. A scene on each key, and the knob dims the lights through a webhook.
  • Editing. Undo and redo on the knob, zoom on push and turn, export on a key.
  • AI coding agents. Approve or interrupt from a key, scroll the output with the knob, and let the agent light a key when it needs you. This is what PadKit was built for.

Shortcut

Let an agent set it up for you.

The repository carries a machine-readable spec covering the input map, the HID protocol for lights and keymap, and the daemon's shell, webhook and MCP actions. Paste this to Claude or any other coding agent, fill in the blank, and it can write your keybindings or a daemon config for you.
